## 一、Alert State TTL（re-fire 機制）

### 背景 / 問題

`broker_alert_state.json` 等五個 alert state 檔案沒有過期機制。一旦某個 condition（如 unfilled orders）發生並發送告警後，如果狀態指紋不變，就永遠不會 re-fire，即使問題持續存在。

### 新增 `app/core/settings.py`

```python
ALERT_REFIRE_SECONDS = _get_int("CRYPTO_ALERT_REFIRE_SECONDS", 86400)
```

預設 24 小時；設為 0 則停用 TTL（舊行為）。

### 新增 `app/alerting/state.py`

集中化 alert state 基礎操作：

| 函式 | 說明 |
|---|---|
| `build_fingerprint(payload)` | SHA-256 雜湊，任何欄位變動即觸發 re-alert |
| `write_alert_state(file, state)` | 寫入並自動附加 `written_at` ISO timestamp |
| `read_alert_state(file, ttl_seconds)` | 若 `age > ttl_seconds` 回傳 `None`，強制 re-alert |
| `clear_alert_state(file)` | 刪除 state 檔案 |

### 修改五個 alert 模組

`broker.py`、`health.py`、`worker.py`、`execution.py`、`queue.py`：

- 移除各自的 `import hashlib`、重複的 fingerprint/state 實作
- 改 import `app.alerting.state` 函式
- `_read_state()` 改傳入 `ttl_seconds=ALERT_REFIRE_SECONDS`

### 新增測試（5 個）

- `write_alert_state` 寫入含 `written_at` 欄位
- `read_alert_state` 在 TTL 過期後回傳 `None`
- `read_alert_state` 在 TTL 未過期時回傳 state
- `read_alert_state` TTL=0 永不過期
- broker alert 在 TTL 過期後 re-fire

## 二、Stage 4 — Backtesting Framework

### 新增 `app/backtest/runner.py`

`run_backtest()` 核心引擎：

- 建立獨立 in-memory SQLite DB（含完整 migrations）
- 按 `open_time` 排序，逐根插入 K 線，逐步呼叫 `strategy_fn → evaluate_signal_id → _simulate_fill`
- 直接插入 order + fill（繞過 broker），呼叫 `update_positions`、`rebuild_daily_realized_pnl`
- 支援 `fill_on="close"`（預設，當根 K 線收盤成交）與 `fill_on="next_open"`（下根開盤，避免 bar-close lookahead bias）
- 回傳：`symbol`, `strategy_name`, `candle_count`, `trade_count`, `metrics`, `equity_curve`, `trades`

**已知限制（docstring 說明）：**
- Cooldown 使用 `datetime.now(utc)`，歷史 K 線永遠通過 → 建議 `cooldown_seconds=0`
- `max_daily_loss > 0` 會觸發 `enable_kill_switch()`（寫 disk）→ 建議保持預設 `0.0`

### 新增 `app/backtest/metrics.py`

`compute_metrics()` 計算：

| 指標 | 說明 |
|---|---|
| `total_return_pct` | (final_equity - initial_capital) / initial_capital |
| `max_drawdown_pct` | 最大回撤百分比 |
| `sharpe_ratio` | 年化 Sharpe（252 交易日，日頻 returns），少於 2 天回傳 `None` |
| `win_rate_pct` | 勝率（BUY/SELL 配對回合） |
| `profit_factor` | 毛利 / 毛損，全勝回傳 `inf` |
| `round_trips` | 完整 BUY→SELL 配對次數 |

## 三、Backtest 擴充：Loader、Parameter Sweep、Walk-Forward

### `app/backtest/loader.py`

`load_candles_from_db(connection, symbol, timeframe, start, end, limit)`：
- 接受 ISO date string（`"2024-01-01"` 或 `"2024-01-01 08:00:00"`），內部轉 epoch_ms
- 支援 `start`（含）/ `end`（不含）/ `limit` 過濾
- 回傳按 `open_time ASC` 排序的 dict list

### `app/backtest/sweep.py`

`run_parameter_sweep(symbol, strategy_name, candles, param_grid, sort_by)`：
- `param_grid` 支援鍵：`order_qty`、`max_position_qty`、`cooldown_seconds`、`max_daily_loss`
- `itertools.product` 枚舉所有組合，逐一呼叫 `run_backtest()`
- `sort_by="max_drawdown_pct"` 升序，其餘降序排列
- 未知參數即時 raise `ValueError`

### `app/backtest/walk_forward.py`

`run_walk_forward(symbol, strategy_name, candles, n_splits)`：
- Expanding window：將 candles 分成 `n_splits + 1` 等份
- Fold i：train = `candles[:chunk*(i+1)]`，test = `candles[chunk*(i+1):chunk*(i+2)]`
- `oos_metrics`：各 fold test 結果的 mean / std（`total_return_pct`、`sharpe_ratio`、`max_drawdown_pct`、`win_rate_pct`）

## 四、Backtest API Endpoints

### `GET /backtest`

```
symbol=BTCUSDT, strategy=ma_cross, days=30 (1-365),
order_qty, max_position_qty, fill_on={close|next_open}, initial_capital
```

從 DB 載入最近 N 天 K 線，回傳 `run_backtest()` 完整結果。

### `POST /backtest/sweep`

```json
{
  "symbol": "BTCUSDT", "strategy": "ma_cross", "days": 30,
  "param_grid": {"order_qty": [0.001, 0.002]},
  "sort_by": "sharpe_ratio"
}
```

回傳所有組合結果（`combination_count`, `results`）。

### `POST /backtest/walk-forward`

```json
{"symbol": "BTCUSDT", "strategy": "ma_cross", "days": 90, "n_splits": 5}
```

回傳 per-fold train/test metrics 及 `oos_metrics` 彙總。

**統一錯誤格式**：未知策略、無 K 線資料、無效參數均回傳 `{"error": "..."}` 而非 4xx。

## 五、新策略：RSI、Bollinger Bands、MACD

### `app/strategy/rsi.py`

- 預設：period=14，oversold=30，overbought=70
- Wilder's smoothing 計算 RSI
- 最少需要 `period + 1` 根 K 線；實際 fetch `period × 3` 以穩定 smoothing
- `short_ma` 儲存 RSI 數值，`long_ma = 0.0`

| 條件 | 訊號 |
|---|---|
| RSI ≤ oversold | BUY |
| RSI ≥ overbought | SELL |
| 其他 | HOLD |

### `app/strategy/bbands.py`

- 預設：period=20，num_std=2.0
- Population std（非 sample）
- `short_ma` 儲存 close，`long_ma` 儲存 middle band

| 條件 | 訊號 |
|---|---|
| close ≤ lower band | BUY（均值回歸） |
| close ≥ upper band | SELL |
| 其他 | HOLD |

### `app/strategy/macd.py`

- 預設：fast=12，slow=26，signal=9
- EMA 使用標準 multiplier `k = 2/(period+1)`
- Fetch `slow × 3 + signal` 根 K 線保證 EMA warmup 品質
- `short_ma` 儲存 MACD line，`long_ma` 儲存 signal line

| 條件 | 訊號 |
|---|---|
| histogram 負轉正（MACD 上穿 signal） | BUY（趨勢追蹤） |
| histogram 正轉負（MACD 下穿 signal） | SELL |
| 其他 | HOLD |

### 修改 `app/strategy/registry.py`

STRATEGY_REGISTRY 新增三個策略（按字母排序）：
