# 2026-03-29 修改紀錄

## Closed Trades 混用 1m / 5m 配對修補

### 問題背景

遠端 scheduler 已切換為：

- `runtime/scheduler.timeframes = 5m`
- `runtime/scheduler.strategy = ppo`

但 Admin UI 的 closed trades 仍出現像是只持有 `1.02 min` 的短單，看起來像 `5m` 模型在 1 分鐘內平倉。

範例：

- `Entry = 66748.24`
- `Exit = 66756.82`
- `Closed At = 2026-03-28 14:58:40`
- `Hold = 1.02 min`

### 根因

這筆其實是舊的 `1m` 交易，不是新的 `5m` 交易。

原本 `/strategies/closed-trades` 的配對邏輯在 `app/query/read_service.py` 只用：

- `strategy_name`
- `symbol`

當作倉位 key，沒有把 `timeframe` 納入。

因此在切換到 `5m` 之後，新的 `5m SELL` 可能會去配對先前未平掉的 `1m BUY`，造成：

- `1m` / `5m` closed trade 被混成同一筆
- UI 看起來像 `5m` 單只持有 1 分鐘

### 修補內容

#### `app/query/read_service.py`

將 closed trade 配對 key 從：

- `(strategy_name, symbol)`

改為：

- `(strategy_name, symbol, timeframe)`

同步修正：

- `get_strategy_closed_trades()`
- execution report / closed trade 統計內部的 entry lookup

現在 BUY / SELL 只會在相同 timeframe 下互相配對。

#### `app/admin/page.py`

Closed trades 卡片標題改為直接顯示：

- `strategy_name · symbol/timeframe`

例如：

- `ppo · BTCUSDT/5m`
- `ppo · BTCUSDT/1m`

避免 UI 誤導。

### 遠端驗證

已同步修補到：

- `root@168.144.35.241:/root/crypto`

並重啟 API 後重新驗證 `/strategies/closed-trades?limit=12`。

修補後結果：

- 最新 closed trades 前 4 筆正確顯示為 `5m`
- 後續舊紀錄才是 `1m`

範例：

- `2026-03-28 23:00:08` → `BTCUSDT`, `5m`
- `2026-03-28 22:40:51` → `BTCUSDT`, `5m`
- `2026-03-28 21:20:42` → `BTCUSDT`, `5m`
- `2026-03-28 20:35:03` → `BTCUSDT`, `5m`
- `2026-03-28 15:21:01` → `BTCUSDT`, `1m`
- `2026-03-28 14:58:40` → `BTCUSDT`, `1m`

這證明：

- `5m` 交易並沒有被 1 分鐘平倉
- 問題來自 closed trades API 的跨 timeframe 混配
- 修補後 `1m` / `5m` 已正確分開
