### 1. Admin Console 分頁化（Tab Navigation）

**問題：** Admin 頁面資料量龐大，需要一直滾動才能到達目標區塊，操作不方便。

**改動：**

- 將 Admin Console 拆分為 6 個分頁：
  - **總覽** — Hero stats、系統狀態、Metrics、系統說明
  - **控制** — Pipeline、Scheduler、Kill Switch
  - **市場資料** — 資料狀態、手動抓取、Feature Store（新分頁）
  - **監控** — Strategy Activity、Queue、Heartbeats、Orders、Positions、Trades、Portfolio、Risk Config
  - **ML / AI** — Training Jobs、Model Registry、Inference、RL Experiment
  - **診斷** — Health Report、PnL、Pipeline Result、Audit、Alerts、Soak Validation
- Feature Store 從 ML/AI 分頁移至市場資料分頁（概念上更符合資料層）

**檔案：** `app/admin/page.py`

### 2. 市場資料層改善

#### 2a. `GET /candles/status` — 新增 Candle 狀態 API

**問題：** 沒有任何 API 可以查看每個 symbol 的 candle 數量、資料新鮮度與缺口狀況。

**改動：** `app/data/candles_service.py` 新增 `get_candles_status()`，回傳每個 `(symbol, timeframe)` 的：
- `count` — candle 總數
- `earliest` / `latest` — 最早與最新時間（UTC ISO 格式）
- `stale_seconds` — 距最新一根 candle 的秒數
- `has_gaps` — 是否偵測到缺口
- `gap_count_estimate` — 估計缺口數量（基於實際時間跨度 vs 預期時間跨度）

`app/api/main.py` 新增 `GET /candles/status` endpoint。

**Admin UI：** 市場資料分頁「資料狀態」區塊，點 Refresh Status 顯示每個 symbol 的狀態，含顏色標示（綠/橙/紅）。

#### 2b. `POST /market-data/fetch` — 獨立觸發市場資料抓取

**問題：** 原本只能透過跑完整 pipeline 來抓取新 candle，無法獨立觸發。

**改動：** `app/api/main.py` 新增 `POST /market-data/fetch`：
- 接受可選的 `symbols` 列表（JSON body）
- 若未傳入，使用 scheduler 的 active symbols
- 直接呼叫 `run_market_data_job()`，不觸發 strategy/risk/execution

**Admin UI：** 市場資料分頁「抓取市場資料」區塊，可填入 symbol 後點 Fetch Now。

### 3. RL + Inference 整合（承接前日工作）

**問題：** RL 訓練完後結果只存在 `training_jobs`，沒有進入 model registry，無法透過 inference service 使用。

**改動：**
- `RLJobRequest` 新增 `auto_promote: bool = False` 參數
- `POST /training/rl-jobs` 訓練成功後自動呼叫 `register_model()` 將 RL agent 存入 model registry（as candidate）
- `auto_promote=True` 時自動呼叫 `promote_model()` 設為 champion
- 回傳新增 `registry_model_id` 和 `registry_status`
- Inference service 無需修改，已支援 RL（與 supervised 共用相同 sigmoid 推論邏輯）
- 新增 6 個測試（Group 5: RL registry integration）

**ML/AI 分頁：** RL Experiment 面板已加入 auto-promote checkbox。

## 已知缺口（尚未處理）

| 項目 | 說明 |
|---|---|
| Retry / backoff | `fetch_klines()` 失敗一次就讓 job failed，無重試邏輯 |
| Feature materialization 未整合進 market_data_job | 每次抓完新 candle 需手動觸發特徵更新 |
| 價格欄位存為 TEXT | `open/high/low/close` 字串，計算時需轉型 |
| Symbol 寫死在程式碼 | 新增交易對需改 `symbols.py` |
| 缺乏 DB index | (symbol, timeframe, open_time) 無獨立 index，大資料量時查詢較慢 |
| 固定 1m timeframe | 不支援 5m / 1h 等多 timeframe |
